# Examples Index

This folder is for staging real project files that Design sessions can reference.
It is gitignored — nothing here is committed.

## How to use

1. Create a subfolder for each category (e.g., `examples/templates/`, `examples/mockups/`)
2. Drop files in (Word docs, PDFs, screenshots, etc.)
3. Write a `notes.md` next to each file — bullet points about what it is and what's notable
4. Keep this manifest current so Design sessions can see what's staged
5. Design sessions will read this manifest and selectively load relevant files
